House Price Prediction - FAQ

1. Why does Lasso perform well in RMSE but very poorly in R²? What is the relationship between these two metrics?
RMSE measures average prediction error, while R² measures how well the model explains variance. Lasso can have a low RMSE if it predicts close to the mean, but if it underfits and doesn't capture variance, R² can be very low or negative. So, a model can have a reasonable RMSE but poor R² if it underfits.
2. What is the main advantage of using Ridge over Lasso for this dataset?
Ridge keeps all features by shrinking coefficients, which helps when many features are relevant. Lasso can remove too many features, leading to underfitting and poor R².
3. What is the main takeaway from your results?
Ridge Regression gave the best balance of accuracy and generalization, with an average error of ~$14,500. The pipeline and features are effective for this problem.
4. What steps did you take to ensure your model generalizes well?
I used feature engineering, regularization, and compared multiple models. I validated on unseen data and plan to add cross-validation and hyperparameter tuning.
5. Why did you include both linear and ensemble models?
Linear models are interpretable and fast. Ensemble models like Random Forest and XGBoost capture complex patterns. Comparing both ensures the best model is chosen based on real performance.
6. How can someone run or deploy your app?
Install dependencies with pip install -r requirements.txt, then run streamlit run app/app.py. The README has full instructions.
7. What are your next steps or future improvements?
I plan to add hyperparameter tuning, model explainability with SHAP, cross-validation, and location-based price maps.
8. What is the main advantage of using log transformation on the target variable?
Log transformation helps stabilize variance and makes the target distribution more normal, which improves model performance and reduces the impact of outliers.
9. Why is model explainability important in this project?
Model explainability (e.g., with SHAP) helps understand which features drive predictions, making the model more transparent and trustworthy for users and stakeholders.
10. How do you handle missing values in your pipeline?
I use median imputation for numeric features and most frequent or 'Missing' for categorical features, ensuring the pipeline can handle real-world data with gaps.
